## ğŸ—‚ï¸ Data Structure: Priority Queue

### Why Priority Queue?

The **Priority Queue** (implemented using a Min-Heap) is perfect for this clinic system because:

1. **Automatic Ordering:** Elements are automatically sorted by priority
2. **Efficient Operations:** O(log n) time complexity for insertion and deletion
3. **Emergency Handling:** Critical cases are automatically prioritized
4. **Fair Treatment:** FIFO ordering within same priority ensures fairness

### Time Complexity

| Operation | Time Complexity | Description |
|-----------|----------------|-------------|
| **Insert** (Add Patient) | O(log n) | Heap insertion with re-heapify |
| **Delete** (Call Next Patient) | O(log n) | Remove root and re-heapify |
| **Update** (Modify Patient) | O(n) | Remove and re-insert patient |
| **Search by ID** | O(1) | Using HashMap for instant lookup |
| **Search by Name** | O(n) | Linear search through all patients |
| **Peek** (View Next) | O(1) | View root without removal |
